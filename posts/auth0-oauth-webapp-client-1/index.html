Create a webapp client app with Auth0 OAuth

Prerequisites#

  • Create an account on https://auth0.com/signup with a free plan
  • Navigate to “Applications -> Applications” to see your “Default App”. Grab the “Client ID”, “Client Secret” and “Domain” from “Settings” page of Default App
  • Navigate to “Applications -> Applications -> Default App -> APIs” and perform the following actions:
    • Turn on authorization for “Auth0 management API”
    • Expand the dropdown to select granular options
      • create:clients (Allows the generated access token to create clients using management API)
      • read:clients (List available clients)
  • Navigate to “Applications -> Applications -> Default App -> Settings -> Application Properties” and select “Machine to Machine” and save the changes

    • Get an access token to make management API calls
      curl -s --request POST --url "https://$DOMAIN/oauth/token" \
--header 'content-type: application/json' \
--data-binary @- <<EOF | jq
{
"client_id": "$CLIENT_ID",
"client_secret": "$CLIENT_SECRET",
"audience":"https://$DOMAIN/api/v2/",

Generate an access token for the webapp client application#

  1. Requesting an authorization code usually requires a user-agent and a callback listener to get the code. Once we request authorization code, server responds with a HTTP 302 response with Location set to your specified callback URL (localhost:3000/callback in our case). On receiving an HTTP response of 302 with a Location, browser redirects to the specified location. In production, this would be a backend API call that is handled by your backend server. Even though the user can see the redirection and the authorization code, the client secret is not available to the user/resource owner assuring us that client is the only entity that can get access tokens.
  2. Lets setup a quick and simple server that listens for HTTP connections on port 3000 locally and shows us the authorization code
    • Install uv to manage packages and project ( What is uv?)
      pip install uv
    • Create a new directory to install packages
      mkdir /tmp/callback-handler; cd /tmp/callback-handler
    • Create a new virtual environment to install packages in the directory

      source .venv/bin/activate
$ uv pip install flask

    • Create a file “server.py” with the following contents
      from flask import Flask, request
app = Flask(__name__)

@app.route("/callback")
def callback():
	code = request.args.get("code")
	if code:
		print(f"Authorization code: {code}")
		return "Success! Close this tab."
	return "No code received.", 400

if __name__ == "__main__":
	app.run(port=3000)
    • Run the server to start listening for connections on port 3000
      uv run server.py
  3. Construct and open the following URL in your browser using the client_id and client_secret received while creating the client
    https://dev-s6oegu3u80i0uqf1.us.auth0.com/authorize?response_type=code&client_id=0hQ3kjyiGa5NTcDtXyaVPPMHE63zguBN&redirect_uri=http://localhost:3000/callback&scope=openid profile email&state=foobar
    • Your browser will show a login screen to use your Auth0 account credentials and a consent screen to approve the requested scopes (openid, profile and email). Approve and consent to the requested scopes.
