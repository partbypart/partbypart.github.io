Quick bytes - Generate a self-signed certificate using openssl

Learn how to generate a self signed certificate using openssl
4 min · 660 words

Prerequisites#

  • Linux/MacOS
  • openssl command line tool (Version 3.x.x)

Self signed certificates can be generated using openssl x509, openssl ca and openssl req commands.

Generate a self signed certificate using openssl req#

  • Generate the self signed certificate
    $ openssl version -v
OpenSSL 3.0.2 15 Mar 2022 (Library: OpenSSL 3.0.2 15 Mar 2022)

$ openssl req -x509 -sha256 -days 1825 -newkey rsa:2048 \
-keyout ca.key.pem \
-out ca.cert.pem \
-subj "/C=XX/ST=State/L=City/O=Company/OU=Section/CN=example.com"

Generate a self signed certificate using openssl x509#

  • Create a configuration variable in bash to add extensions
    $ CONFIG="
[extensions]
subjectKeyIdentifier = hash
authorityKeyIdentifier = keyid:always
basicConstraints=critical, CA:TRUE"
  • Generate an RSA private key to sign the certificate (cannot be done using openssl x509)
    $ openssl genpkey -algorithm rsa -pkeyopt rsa_keygen_bits:4096 -out ca.key.pem
  • Generate a self signed certificate using the extensions specified in $CONFIG bash variable
    $ openssl x509 -new -key ca.key.pem -out ca.cert.pem \
-subj "/C=XX/ST=State/L=City/O=Company/OU=Section/CN=example.com" \
-days 1825 -CAcreateserial \
-extfile <(echo "$CONFIG") \
-extensions extensions
  • Verify the self signed certificate data by dumping relevant information from certificate
    $ openssl x509 -in ca.cert.pem  -noout -text \
-certopt no_issuer -certopt no_sigdump \
-certopt no_pubkey -certopt no_serial \
-certopt no_version -certopt no_signame

### Generate a self signed certificate using openssl ca#

  • openssl ca cannot generate a self signed certificate by itself in a single command. It can sign a CSR (Certificate Sign Request) using the key used in CSR (self sign)
  • openssl ca expects a configuration file with a few mandatory fields under [ ca ] section to sign a request
  • Let’s generate a CSR using an existing private key
    $ openssl req -new -key ca.key.pem -out ca.csr.pem \
-subj "/C=XX/ST=State/L=City/O=Company/OU=Section/CN=example.com"
$ ls ca.csr.pem
ca.csr.pem
  • Let’s use a sample minimal CA configuration
    $ cat openssl.cnf
[ ca ]
default_ca = CA_default

[ CA_default ]
dir               = /home/alice/ca
certs             = $dir/certs
new_certs_dir     = $dir/newcerts
database          = $dir/index.txt
serial            = $dir/serial
private_key       = $dir/private/ca.key.pem
default_md        = sha256
name_opt          = ca_default
cert_opt          = ca_default
default_days      = 1875
preserve          = no
policy            = policy_strict

[ policy_strict ]
# The root CA should only sign intermediate certificates that match.
# See the POLICY FORMAT section of `man ca`.
countryName             = match
stateOrProvinceName     = match
organizationName        = match
organizationalUnitName  = optional
commonName              = supplied

[ v3_ca ]
subjectKeyIdentifier = hash
authorityKeyIdentifier = keyid:always,issuer
basicConstraints = critical, CA:true
keyUsage = critical, digitalSignature, cRLSign, keyCertSign
  • Self-sign the CSR using the configuration from openssl.cnf
    $ openssl ca -config openssl.cnf -selfsign \
-key ca/private/ca.key.pem \
-out ca/certs/ca.cert.pem \
-in ca.csr.pem \
-extensions v3_ca
  • Verify certificate contents
    $ openssl x509 -in ca/certs/ca.cert.pem  -noout -text \
-certopt no_issuer -certopt no_sigdump \
-certopt no_pubkey -certopt no_serial \
-certopt no_version -certopt no_signame
